Intro a aplicaciones Web con Java

Breve historia, definición e importancia de las aplicaciones web.

Evolución de las Aplicaciones Web

Las aplicaciones web han evolucionado desde simples páginas HTML estáticas, ofreciendo contenido fijo, a complejas aplicaciones dinámicas capaces de proporcionar experiencias personalizadas a los usuarios. La introducción de AJAX a principios de los 2000 marcó un punto de inflexión, permitiendo desarrollos web más interactivos y reactivos.

Aplicaciones Web Estáticas

Las aplicaciones web estáticas sirven contenido fijo a todos los usuarios. Están construidas principalmente con HTML, CSS, y JavaScript básico. Son rápidas, seguras y fáciles de desplegar, ideales para sitios informativos, blogs y portafolios.

Se pueden construir a mano y/o usando frameworks y librerías.

Aplicaciones Web Dinámicas

A diferencia de las estáticas, las aplicaciones web dinámicas generan contenido en tiempo real basado en las interacciones del usuario. En el contexto de Java, se utilizan tecnologías como Servlets, JavaServer Pages (JSP), y frameworks como Spring y JavaServer Faces (JSF) para manejar la lógica del lado del servidor.

Aplicaciones Web Dinámicas

Estas tecnologías permiten desarrollar aplicaciones web robustas y escalables que pueden gestionar complejas transacciones y dinámicas de usuario. Del lado del cliente, también se pueden integrar con tecnologías como Angular, React, Vue.js y Lit para crear interfaces de usuario ricas e interactivas.

Importancia de las Aplicaciones Web

Las aplicaciones web son fundamentales en la era digital, impulsando el comercio electrónico, las redes sociales, y plataformas de servicios. Su capacidad para ofrecer contenido dinámico y personalizado, accesible desde cualquier dispositivo con acceso a internet, ha transformado la manera en que interactuamos con la tecnología.

Resumen y algo de Contenido Extra

Iniciar Tomcat

  1. Ejecuta el script startup.sh (Unix/Linux) o startup.bat (Windows) en el directorio bin/.
  2. Verifica que Tomcat esté corriendo accediendo a http://localhost:8080.

Acceder a la Aplicación Desplegada

Una vez desplegada, accede a tu aplicación web navegando a http://localhost:8080/nombreDeTuAplicacion, sustituyendo nombreDeTuAplicacion por el nombre de tu archivo WAR sin la extensión .war.

Acceso a Logs del Servidor

Los logs de Tomcat son esenciales para depurar tu aplicación:

  • Encuentra los logs en el directorio logs/ dentro de tu instalación de Tomcat.
  • El archivo catalina.out generalmente contiene toda la salida del servidor.
  • Utiliza herramientas de línea de comandos o editores de texto para revisar estos archivos y diagnosticar problemas.

Para entornos Unix/Linux, puedes usar tail -f catalina.out para ver el log en tiempo real.

Detener Tomcat

  1. Ejecuta el script shutdown.sh (Unix/Linux) o shutdown.bat (Windows) en el directorio bin/ de tu instalación de Tomcat.
  2. Para verificar que Tomcat se ha detenido correctamente, puedes revisar los logs en el directorio logs/, buscando mensajes de confirmación de detención o intentar acceder nuevamente a http://localhost:8080. Si Tomcat se detuvo correctamente, la página no debería cargar.

Documentación Oficial de Tomcat

Para una comprensión profunda y recursos adicionales sobre Apache Tomcat, se recomienda consultar la documentación oficial:

  • Visita la Documentación Oficial de Tomcat 9 para obtener guías detalladas, configuración, y mejores prácticas.
  • La documentación cubre desde la instalación básica hasta conceptos avanzados de seguridad y tuning de rendimiento.

Configuración de Tomcat mediante Archivos XML

Apache Tomcat permite una amplia configuración mediante el uso de archivos XML. Los más importantes incluyen:

  • server.xml: El archivo principal de configuración del servidor, utilizado para configurar componentes como conectores, engines, y hosts.
  • web.xml: Ubicado en el directorio WEB-INF/ de tu aplicación, define la configuración específica de la aplicación, como servlets, filtros, y listeners.
  • context.xml: Define la configuración de contexto para aplicaciones web, incluyendo recursos como DataSources y JNDI.

Configuración de Tomcat mediante Archivos XML

Editar estos archivos permite personalizar el comportamiento del servidor y las aplicaciones desplegadas en él. Se recomienda hacer backup antes de hacer cambios significativos.

Más información en la sección de configuración de la documentación oficial de Tomcat.

Configuración de Conectores en server.xml

Los conectores en Tomcat definen cómo el servidor se comunica con el mundo exterior. A través de la configuración en server.xml, puedes:

  • Modificar el puerto HTTP/HTTPS por defecto.
  • Configurar SSL/TLS para mejorar la seguridad.
  • Ajustar parámetros de rendimiento como el tamaño del thread-pool.

Estos ajustes son cruciales para el rendimiento y la seguridad de tu servidor web.

Configuración de Contextos en context.xml

El archivo context.xml permite definir cómo Tomcat maneja las aplicaciones web individuales, incluyendo:

  • Gestión de recursos como JNDI DataSources para bases de datos.
  • Configuración de parámetros de contexto que afectan al ciclo de vida de la aplicación.
  • Control de opciones de despliegue como el path de la aplicación y la configuración de sesión.

Personalizar el contexto de una aplicación mejora su integración con el sistema y optimiza su comportamiento.

Seguridad y Autenticación mediante web.xml

La configuración de seguridad de las aplicaciones web se realiza en el archivo web.xml, permitiendo:

  • Definir roles y restricciones de seguridad para controlar el acceso a recursos web.
  • Configurar métodos de autenticación como BASIC, DIGEST, FORM, o CLIENT-CERT.
  • Establecer parámetros de sesión y cookies para gestionar la identificación de usuarios.

Configurar adecuadamente la seguridad es fundamental para proteger la aplicación y los datos de los usuarios.

Optimización del Rendimiento

A través de la modificación de archivos XML, se pueden implementar optimizaciones de rendimiento, tales como:

  • Ajustes de cache y timeout para conectores y contextos.
  • Configuración de compresión para reducir el tamaño de las respuestas HTTP.
  • Optimización de recursos JNDI y pools de conexiones para bases de datos.

Estas optimizaciones pueden significar una mejora considerable en la capacidad de respuesta y la eficiencia de tu servidor y aplicaciones web.

Tecnologías para Aplicaciones Web Dinámicas

Las aplicaciones web dinámicas permiten interacciones personalizadas con los usuarios en tiempo real. Para su desarrollo, se pueden utilizar múltiples tecnologías:

  • Java Servlets y JavaServer Pages (JSP)
  • PHP
  • ASP.NET
  • JavaScript con Node.js
  • Frameworks como Spring (Java), Django (Python), y Ruby on Rails (Ruby)

Estas tecnologías permiten al servidor generar contenido dinámico basado en las acciones y solicitudes de los usuarios.

Objetivos

Por razones de compatibilidad con Tomcat en este minicurso, nos centraremos en los Servlets y las páginas JSP.

Servlets

¿Qué son los Servlets?

Los Servlets son clases Java que manejan solicitudes y respuestas en aplicaciones web. Operan en el lado del servidor para:

  • Leer datos enviados por el usuario (formularios, cookies, etc.)
  • Procesar esos datos
  • Generar y enviar respuestas dinámicas al cliente

Forman la base de muchas tecnologías Java EE, incluyendo JSP y frameworks como Spring y JSF.

Desarrollo de Servlets

Para desarrollar un Servlet, se deben seguir estos pasos básicos:

  1. Crear una clase que extienda HttpServlet.
  2. Sobrescribir los métodos doGet o doPost para manejar solicitudes GET o POST.
  3. Compilar el Servlet y desplegarlo en un contenedor de Servlets como Tomcat.

Los Servlets pueden interactuar con otros componentes web, como JSP, para generar respuestas complejas.

Ciclo de Vida de un Servlet

El ciclo de vida de un Servlet comprende tres etapas principales:

  • Inicialización: El contenedor de servlets carga el servlet y llama al método init().
  • Procesamiento de Solicitudes: El contenedor invoca los métodos doGet(), doPost(), etc., según el tipo de solicitud.
  • Finalización: El contenedor de servlets retira el servlet del servicio llamando al método destroy() y luego libera cualquier recurso asociado al servlet.

Este ciclo permite manejar eficientemente múltiples solicitudes a lo largo de la vida útil del servlet. Nosotros apenas tocamos este aspecto de la tecnología, pero hay que conocerla.

Despliegue y Ejecución de Servlets

Para desplegar y ejecutar un Servlet en un contenedor como Tomcat, se deben realizar los siguientes pasos:

  1. Empaquetar el Servlet y cualquier recurso necesario en un archivo WAR.
  2. Desplegar el archivo WAR en el contenedor de servlets colocándolo en el directorio webapps.
  3. Configurar el servlet en el archivo web.xml o mediante anotaciones en el código del Servlet.

Una vez desplegado, el Servlet puede manejar solicitudes enviadas a su URL mapeada.

Páginas JSP

Introducción a JavaServer Pages (JSP)

JavaServer Pages (JSP) es una tecnología que permite a los desarrolladores crear contenido web dinámico y generar páginas HTML dinámicamente. JSP se ejecuta en el servidor y combina HTML estándar con elementos Java y JSP específicos para producir páginas web que responden a las solicitudes de los usuarios.

Características Principales de JSP

JSP ofrece varias características poderosas para el desarrollo web, incluyendo:

  • Uso de etiquetas personalizadas para reutilizar componentes y simplificar el código.
  • Capacidad para incrustar código Java directamente en páginas HTML.
  • Soporte para patrones de diseño MVC (Modelo-Vista-Controlador), facilitando el desarrollo de aplicaciones web robustas y mantenibles.
  • Integración con tecnologías Java EE como Servlets, JDBC para acceso a bases de datos, y JNDI para acceso a recursos empresariales.

Desarrollo de Páginas JSP

Crear una página JSP implica:

  1. Crear un archivo con extensión .jsp que contiene HTML junto con elementos JSP y código Java.
  2. Utilizar etiquetas JSP como <jsp:directive>, <jsp:scriptlet>, y <jsp:expression> para controlar el comportamiento de la página.
  3. Desplegar la página JSP en un contenedor de servlets/JSP como Apache Tomcat para su ejecución.

Las páginas JSP simplifican el proceso de creación de contenido dinámico, permitiendo a los desarrolladores centrarse en la lógica de la aplicación.

Ciclo de Vida de una Página JSP

El ciclo de vida de una página JSP incluye:

  • Traducción: El archivo JSP se traduce a un servlet Java la primera vez que se solicita.
  • Compilación: El servlet Java se compila en un archivo de clase ejecutable por el servidor.
  • Ejecución: El servidor ejecuta el servlet resultante, procesando las solicitudes y generando la salida enviada al cliente.
  • Finalización: Opcionalmente, el servlet puede ser destruido por el servidor, liberando recursos.

Este proceso permite que las páginas JSP sean modificadas y actualizadas fácilmente sin necesidad de recompilar manualmente el código Java.

Despliegue y Ejecución de Páginas JSP

Para desplegar y ejecutar una página JSP:

  1. Coloca el archivo .jsp en el directorio web-appropriate de tu servidor (por ejemplo, el directorio webapps en Tomcat).
  2. Accede a la página JSP a través de un navegador web usando la URL correspondiente al archivo JSP dentro del servidor.
  3. El servidor procesará la página JSP, ejecutará cualquier código Java incrustado y devolverá el resultado como una página HTML al cliente.

Las páginas JSP permiten una forma eficiente de crear y servir contenido web dinámico, aprovechando la potencia y la portabilidad de Java.

Integración de Lit con Aplicaciones Web Dinámicas

Lit es una biblioteca ligera y eficiente para construir componentes web reactivos. Integrar Lit en aplicaciones web dinámicas permite crear interfaces ricas que interactúan con el servidor de manera eficaz, aprovechando las rutas de los servlets para solicitudes de datos y operaciones.

Lit a lo bestia

Construcción de una Aplicación Lit

El proceso de construcción convierte tu código de desarrollo en una versión optimizada para producción. Para una aplicación Lit, sigue estos pasos:

  1. Utiliza herramientas de construcción como Webpack, Rollup, o Vite. Estas herramientas ayudan a empaquetar y minimizar tu código, además de resolver dependencias.
  2. Configura tu herramienta de construcción para transpilar el código JavaScript moderno a una versión compatible con la mayoría de los navegadores.
  3. Incluye los polyfills necesarios de Lit para asegurar la compatibilidad con navegadores que no soportan Web Components nativamente.
  4. Ejecuta el script de construcción definido en tu package.json, por ejemplo, npm run build, para generar los archivos estáticos.

Esto producirá un conjunto de archivos en tu directorio dist o build, listos para ser desplegados en el servidor.

Desplegando una Aplicación Lit en Tomcat

Una vez construida tu aplicación Lit, el siguiente paso es desplegarla en un servidor como Tomcat:

  1. Copia los archivos generados en el directorio de construcción (dist o build) a la carpeta webapps de tu servidor Tomcat. Dentro de webapps, puedes colocar tu aplicación en un subdirectorio con el nombre de tu aplicación, por ejemplo, webapps/miAppLit.
  2. Reinicia Tomcat si es necesario. Algunas versiones de Tomcat detectan automáticamente los nuevos archivos y los despliegan sin necesidad de reinicio.
  3. Accede a tu aplicación Lit navegando a http://localhost:8080/miAppLit.

Este método te permite servir tu aplicación Lit como archivos estáticos desde Tomcat, mientras que cualquier interacción dinámica con el servidor puede manejarse a través de llamadas a APIs o servlets desde tus componentes Lit.

Lit con JSP

Construcción de un Componente Web con Lit

Imagina que hemos creado un componente web Lit llamado mi-componente, que tiene propiedades como nombre y edad. Este componente es capaz de mostrar información de usuario de manera dinámica.

Empaquetado y Despliegue del Componente

Una vez construido y probado, el componente mi-componente se empaqueta en un archivo JavaScript. Este archivo se debe incluir en la carpeta webapp de nuestra aplicación web, accesible para nuestras páginas JSP.

Integración del Componente en JSP

Para utilizar mi-componente en una página JSP, incluimos el script del componente en el <head> o al final del <body> de nuestra página JSP:

 
  
  <script src="ruta/a/mi-componente.js"></script>

Luego, podemos usar el componente directamente en el HTML:

 
  
  <mi-componente nombre="John Doe" edad="30"></mi-componente>

Uso de Propiedades del Componente en JSP

Las propiedades del componente se pueden establecer dinámicamente en JSP usando expresiones JSP o JSTL. Por ejemplo, para pasar valores dinámicos a nuestro componente:

 
  
  <mi-componente nombre="${usuario.nombre}" edad="${usuario.edad}"></mi-componente>

Donde usuario es un objeto en el modelo de solicitud JSP con atributos nombre y edad.

Interacción con el Componente desde JSP

Además de mostrar datos, podemos interactuar con nuestro componente Lit desde JSP. Por ejemplo, podemos escuchar eventos del componente y actuar en consecuencia:

 
  
    <script>
    document.addEventListener('DOMContentLoaded', function() {
        document.querySelector('mi-componente').addEventListener('miEvento', function(e) {

            alert('Evento recibido con datos: ' + e.detail);
        });
    });
    </script>

Esto permite una interactividad rica entre el componente Lit y la lógica de la página JSP.

Manejo de Eventos Personalizados en JSP

Para reaccionar a eventos personalizados disparados por tu componente Lit en una página JSP, puedes agregar un manejador de eventos directamente dentro de un script en la página:

 
  
    <script>
    document.querySelector('mi-componente-lit').addEventListener('mi-evento-personalizado', function(e) {

        alert('Evento personalizado recibido: ' + e.detail.mensaje);

    });
    </script>

Este script se ejecutará cuando se cargue la página JSP, agregando un escuchador para el evento personalizado, permitiéndote realizar acciones como actualizar la interfaz de usuario o enviar datos al servidor en respuesta al evento.

Delegación de Eventos para Componentes Lit en JSP

Maneja múltiples eventos de componentes Lit agregando un único escuchador de eventos a un contenedor padre:

 
  
    <div id="contenedor-componentes">

      <mi-componente-lit id="componente1"></mi-componente-lit>

      <mi-componente-lit id="componente2"></mi-componente-lit>

    </div>

    <script>

      document.getElementById('contenedor-componentes').addEventListener('mi-evento-personalizado', function(e) {

        if(e.target.id === 'componente1') {

          // Maneja el evento para el componente 1

        } else if(e.target.id === 'componente2') {

          // Maneja el evento para el componente 2

        }

      });

    </script>

Este método permite un manejo eficiente y centralizado de eventos, facilitando la interacción con múltiples componentes Lit en tu JSP.
